# カラースキーマ設計書

## 概要

ProjectLensアプリケーションのカラースキーマは、Backlogのブランドカラーをベースにした「パターンA: クラシック・バックログ」を採用しています。

## 採用パターン: パターンA (Classic Backlog)

BacklogのWeb UIに馴染みのある、信頼感と落ち着きのある配色です。

### ライトテーマ

| カテゴリ | カラーコード | 用途 |
|---------|-------------|------|
| **Primary** | `#2C9A7A` | メインアクション、ヘッダー、ブランドカラー |
| **Secondary** | `#F4F5F7` | 背景、カード背景 |
| **Accent** | `#E83929` | 通知、エラー、強調ボタン |
| **Surface** | `#FFFFFF` | カード、ダイアログ |
| **Background** | `#FFFFFF` | アプリケーション背景 |
| **Error** | `#E83929` | エラーメッセージ |

### ダークテーマ

| カテゴリ | カラーコード | 用途 |
|---------|-------------|------|
| **Primary** | `#4DB699` | メインアクション、ヘッダー（視認性向上のため明るめ） |
| **Secondary** | `#262626` | 背景、カード背景 |
| **Accent** | `#FF6B6B` | 通知、エラー、強調ボタン |
| **Surface** | `#333333` | カード、ダイアログ |
| **Background** | `#121212` | アプリケーション背景 |
| **Error** | `#FF6B6B` | エラーメッセージ |

## デザイン原則

### 1. Backlogとの一貫性
- メインカラー `#2C9A7A` はBacklogのブランドカラー（ティールグリーン）を使用
- 既存のBacklogユーザーにとって違和感のない配色

### 2. アクセシビリティ
- ダークモードでは視認性を確保するため、Primaryカラーを `#4DB699` に明るく調整
- テキストとバックグラウンドのコントラスト比を確保

### 3. 視覚的フィードバック
- ホバー時のカード背景色: `rgba(var(--v-theme-primary), 0.08)`
- ホバー時の影: `0 4px 12px rgba(0, 0, 0, 0.15)`
- ホバー時の動き: `translateY(-2px)`

## 実装ファイル

カラースキーマの設定は以下のファイルで定義されています：

```
src/plugins/vuetify.ts
```
