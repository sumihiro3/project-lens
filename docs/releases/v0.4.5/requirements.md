# ProjectLens レポート/サマリー(生成ダイジェスト) 要件定義書(v0.4.5 ドラフト)

> **リリース**: v0.4.5(複数プロジェクト横断サマリ + 週次/月次レポート + 課題の背景/決定事項要約)
> **ステータス**: ドラフト(着手時に `/refine-requirements v0.4.5` で確定させること)
> **前提**: v0.4(ローカル RAG 検索基盤 + コメント差分取得)の完了。コメントデータは v0.4 で収集済みのものを再利用する。
> 元ドラフト: 旧 v0.4「AIユースケース拡充」のうち**生成ダイジェスト系**を分離したもの(検索基盤=RAG は v0.4 で確定)。

## 目的

v0.3 の FoundationModels 生成基盤と v0.4 で収集したコメントデータの上に、**専用「レポート/サマリー」画面**を新設し、プロジェクト単位・横断の生成ダイジェストを提供する。1日1回のバックグラウンド生成を基本とし、UI を開いたときには結果が準備済みの状態にする。

## 候補機能(壁打ちで v0.4 から移送)

- **複数プロジェクト横断サマリ**(ProjectLens 最大の差別化): 有効な全プロジェクトを横断した状況把握・分析を 1日1回生成。「今週注意すべきこと」を横断で提示。
- **週次/月次アクティビティレポート**: 期間内の更新・完了・新規課題を集約してレポート生成(プロジェクト単位/横断どちらも)。
- **課題の背景・決定事項の要約**: コメントを読んで経緯・決定を要約(v0.4 で取得済みのコメントを利用。新規 API 呼び出しは不要)。課題詳細での per-issue 提供を想定。

## UI(確定方針・壁打ちより)

- 専用「**レポート/サマリー**」画面(新タブ/ページ)を新設し、横断サマリ・週次/月次レポートを集約する。ダッシュボードを肥大化させない。

## 確定時に詰めること

- 横断サマリのコンテキスト上限対策(多数課題の集約 → map-reduce: 課題要約→要約の要約、または SQL 事前集約)。3B 級モデルの上限を考慮。
- 背景要約の埋め込みダイジェストとの共有(v0.4 の未解決事項「埋め込み戦略」で課題ダイジェスト生成を選んだ場合、生成物を再利用できる)。
- 1日1回バッチの実行タイミング(アプリ起動時/前回生成からの経過時間)と手動「再生成」導線。
- レポートの保存・履歴(過去レポートの閲覧)とエクスポート要否。
- 出力言語(UI 言語追従)と海外メンバー向け英語版の要否。
