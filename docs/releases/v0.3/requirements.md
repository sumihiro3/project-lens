# ProjectLens AI機能基盤 要件定義書(v0.3)

> **リリース**: v0.3(macOS 搭載 AI モデルによる推論基盤 + 初回ユースケース)
> **ステータス**: 確定(2026-06-12、`/refine-requirements` での壁打ちにより確定)
> 元ドラフト: AI機能 追加要件定義書(2026-06-12)を3リリースに分割した第1弾。
> ユースケース拡充・MLX バックエンド追加は `../v0.4/requirements.md`、課金・ライセンスは `../v0.5/requirements.md` を参照。

## 背景と目的

Backlog AIアシスタント(プレミアム/プラチナ限定)に相当する機能を、ProjectLens 上で**ローカルAI**により実現する。差別化ポイントは「ローカル実行・外部送信なし」「複数スペース/複数プロジェクト横断」「全Backlogプランで利用可能」。

v0.3 では **macOS 搭載の AI モデル(Apple FoundationModels フレームワーク)** を推論バックエンドとして採用し、最初のユースケース(チケット1行要約・遅延リスク検知)を**最速で届ける**。

- モデルのダウンロード・管理が不要(OS が提供)→ 初回体験が軽い(数GBのDLなし)
- 完全オンデバイス・外部送信なし(差別化ポイント維持)
- guided generation により構造化出力が保証される
- 制約: macOS 26(Tahoe)+ Apple Intelligence 有効環境限定 / 約3Bパラメータ級の品質

旧ドラフトで検討した MLX(Qwen3)バックエンドは**廃止ではなく後続リリース(v0.4 以降)に移動**し、古い macOS のカバーと品質向上の選択肢として追加する。`LlmInference` trait の抽象化はそのための前提として v0.3 で実装する。

課金実装(v0.5)までは全面無料開放とし、フィードバックを集める。

## アーキテクチャ方針(確定)

リアルタイム推論ではなく**バックグラウンド処理**を基本とする。sync のタイミングで裏で LLM 処理を行い、結果を SQLite に保存。UI を開いたときには結果が準備済みの状態にする。

```
Backlog API sync
    ↓
SQLite (issues)
    ↓ 新規・更新チケットを検出 → job_queue に投入
バックグラウンドワーカー(同時1推論)
    ↓ LLM 推論 (FoundationModels / Swift sidecar)
SQLite (ai_results)
    ↓
Vue UI が読むだけ(高速・即時表示)
```

## 機能要件

### FR-V03-001: LLM 推論基盤(`LlmInference` trait + FoundationModels sidecar)

- **優先度**: 必須
- **説明**: バックエンド差し替え可能な抽象化の上に、macOS 搭載モデル(FoundationModels)の推論を実装する
- **受け入れ基準**:
  - `src-tauri/src/ai/` モジュールを新設し、`LlmInference` trait でバックエンドを抽象化する(将来の MLX / Candle 追加はレジストリ登録のみで済む設計)
  - FoundationModels を呼び出す**小さな Swift sidecar**(MLX なし)を JSON Lines over stdin/stdout の常駐プロセスとして実装し、`tauri.conf.json` の `externalBin` で同梱する
  - 構造化出力には FoundationModels の guided generation(`@Generable` 相当)を使い、スキーマ準拠のレスポンスを保証する
  - sidecar の異常終了を検知して自動再起動する(連続失敗時は AI 機能を一時停止しエラー表示)
  - リリースビルド(build.sh)で sidecar が署名・同梱されること

### FR-V03-002: 環境検出と可用性チェック

- **優先度**: 必須
- **説明**: macOS 搭載モデルが使える環境かを判定し、状態を明示する
- **受け入れ基準**:
  - 起動時に可用性を判定する: macOS 26 以上 / Apple Intelligence 有効 / モデル利用可能(FoundationModels の availability API)
  - 利用可能: AI 機能を有効化できる(FR-V03-003)
  - 利用不可: 理由別のメッセージを表示する(「macOS 26 以降が必要」「設定で Apple Intelligence を有効にしてください」等)。Apple Intelligence の設定画面への導線を付ける
  - 非対応環境でも既存機能(同期・一覧・ダッシュボード・通知)はすべて動作する
  - 将来バックエンド(MLX)追加時に「この環境では別のバックエンドを利用できます」と案内できる構造にしておく(表示は v0.4)

### FR-V03-003: AI 機能の有効化フロー

- **優先度**: 必須
- **説明**: オプトイン方式。モデルDLが不要になったため軽量なフローとする
- **受け入れ基準**:
  - 初回起動時(可用性ありの場合)に AI 機能の案内を表示する(外部送信なしの説明つき)。**スキップ可能**
  - スキップした場合、ダッシュボードに「AI を試す」バナーを表示し、設定画面からいつでも有効化できる
  - 設定画面に AI 機能のオン/オフトグルと動作状況(可用性・キュー残件数)を表示する
  - v0.3〜v0.4 の間は全面無料開放(回数制限なし)。Free/Pro ゲートは v0.5 で導入する

### FR-V03-004: バックグラウンド処理基盤(job_queue / ai_results)

- **優先度**: 必須
- **説明**: sync と連動するジョブキューとワーカー
- **受け入れ基準**:
  - SQLite に `ai_results`(issue_id, summary, risk_level, delay_days, suggestion, processed_at, model_used)と `job_queue`(issue_id, job_type, status, created_at)を追加する
  - 処理トリガー4種: sync 直後(新規・更新チケットを投入) / アプリ起動時(未処理キュー再開) / 定期実行 / 手動「再分析」ボタン
  - 対象範囲: **全同期チケットの差分**(初回のみ全件、以降は新規・更新分のみ)
  - 同時推論は1件まで。sync・UI 操作をブロックしない
  - キュー処理状況(残件数・処理中)を設定画面で確認できる

### FR-V03-005: 初回ユースケース(チケット1行要約 + 遅延リスク)

- **優先度**: 必須
- **説明**: チケット単位の AI 分析。SQLite 側で前処理・絞り込みしてから LLM に渡す(コンテキスト対策)
- **受け入れ基準**:
  - チケットごとに生成: 1行要約 / リスクレベル(high・medium・low) / 遅延日数(SQL 計算) / 対応提案
  - 出力は guided generation による構造化で受け取る。生成失敗時はリトライ(上限あり)の上、失敗チケットはスキップして記録する
  - 入力はコンテキスト上限を考慮して前処理する(タイトル・本文先頭・期限・ステータス等に絞り、本文は文字数上限で切り詰め)
  - 出力言語は **UI 言語(ja/en)に追従**する。言語切替時は既存結果を再生成せず、次回処理分から反映する
  - 遅延日数・期限切れ判定は LLM ではなく SQL で算出する(確実性優先)

### FR-V03-006: AI 結果の UI 表示

- **優先度**: 必須
- **説明**: 3箇所に表示する(壁打ちで確定)
- **受け入れ基準**:
  - **IssueCard**: AI 1行要約とリスクバッジ(色分け)を組み込み表示する。AI 結果未生成のチケットでは表示しない(レイアウト崩れなし)
  - **ダッシュボード**: 「遅延リスク課題」セクションを新設し、リスク順に対応提案付きで表示する
  - **課題詳細ダイアログ(新設)**: リスクバッジまたは専用ボタンの押下で開き、AI 要約・対応提案の全文を表示する。既存のブラウザで開く動線は維持する
  - すべての AI 表示に AI 生成であることを示すアイコン/ラベルを付ける

## 非機能要件

### NFR-V03-001: プライバシー(ローカル完結)

- チケットデータを外部(クラウド LLM API 等)に送信しない。推論は FoundationModels のオンデバイスモデルで完結する(Private Cloud Compute は使用しない)

### NFR-V03-002: 動作環境

- AI 機能: macOS 26(Tahoe)以上 + Apple Intelligence 有効(Apple Silicon)
- 既存機能: 従来どおり(AI 非対応環境でも全機能動作)

### NFR-V03-003: リソース消費

- 推論はバックグラウンドで同時1件。UI 応答性(100ms)を維持する
- アイドル時(キュー空)に sidecar が CPU を消費しない

### NFR-V03-004: 堅牢性

- Apple Intelligence が途中で無効化された場合もアプリ本体はクラッシュせず、AI 機能のみ停止して案内を表示する

## スコープ外(やらないこと)

- **MLX(Qwen3)バックエンド** → v0.4 以降(古い macOS のカバー・品質向上の選択肢として追加。trait 抽象化は v0.3 で準備済み)
- モデルのダウンロード・ティア選択・進捗 UI → MLX バックエンド追加時(v0.4 以降)に実装
- 課金・ライセンス・Free/Pro ゲート → v0.5
- ユースケース拡充(プロジェクト横断サマリー・週次/月次レポートなど) → v0.4
- Windows 対応(Candle)、Backlog API への書き込み → 将来
- Gemma 4 対応 → MLX バックエンド導入後の検討事項
- Private Cloud Compute の利用(オンデバイス完結を優先)

## 未解決事項(実装中に判断)

- FoundationModels のコンテキスト上限の実測と、チケット本文の切り詰め文字数の決定
- 約3B級モデルでの日本語要約・提案の品質確認(不足する場合は v0.4 の MLX バックエンド優先度を上げる)
- sidecar のビルド要件(Xcode / macOS 26 SDK)と署名・notarization 手順の build.sh への組み込み
- 開発機・検証機の macOS 26 / Apple Intelligence 環境の確認
- `ai_results` の再分析時の上書きポリシー(バックエンド追加時に全件再生成するか)
